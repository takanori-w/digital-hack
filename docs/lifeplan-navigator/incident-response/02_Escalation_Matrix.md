## 1. エスカレーション概要

### 1.1 目的
インシデント発生時に、適切な担当者・意思決定者へ迅速にエスカレーションするための基準と手順を定義する。

### 1.2 原則
- **迅速性**: 重大度に応じた時間内にエスカレーション
- **正確性**: 必要十分な情報を伴ってエスカレーション
- **適切性**: 適切なレベルの担当者にエスカレーション
- **追跡性**: 全てのエスカレーションを記録

| インシデントタイプ | 初動担当 | 必須エスカレーション先 | 任意エスカレーション先 |
|------------------|----------|----------------------|----------------------|
| **データ漏洩（大量）** | SOC | CSIRT Leader, CISO, CEO, CLO, CFO | CMO |
| **データ漏洩（限定的）** | SOC | CSIRT Leader, CISO, CLO | - |
| **DDoS攻撃** | SOC | CSIRT Leader, Network Eng | CISO（長時間継続時） |
| **ランサムウェア** | SOC | CSIRT Leader, CISO, CEO, CLO, CFO | CMO |

| **内部不正** | SOC | CSIRT Leader, CISO, CLO, HR | CEO（役員関与時） |
| **Webアプリ攻撃** | SOC | CSIRT Eng, App Engineer | CSIRT Leader（被害時） |
| **不正ログイン** | SOC | CSIRT Eng | CSIRT Leader（大量時） |

## 3. エスカレーションフロー

### 3.1 P0（Critical）フロー

```
[検知: SOC Analyst]
        │
        ↓ 即時
[技術対応: CSIRT Engineer + Network Engineer]
        │
        ↓ 即時
[指揮: CSIRT Team Leader]
        │
        ↓ 15分以内
[経営報告: CISO]
        │
        ↓ 30分以内
[最終判断: CEO]
        │
        ├──→ [法務対応: CLO]
        ├──→ [財務対応: CFO]
        └──→ [広報対応: CMO]
```

### 3.2 P1（High）フロー

```
[検知: SOC Analyst]
        │
        ↓ 30分以内
[技術対応: CSIRT Engineer]
        │
        ↓ 30分以内
[指揮: CSIRT Team Leader]
        │
        ↓ 4時間以内（または封じ込め完了時）
[経営報告: CISO]
        │
        ↓ 必要に応じて
[最終判断: CEO]
```

### 3.3 P2（Medium）フロー

```
[検知: SOC Analyst]
        │
        ↓ 4時間以内
[技術対応: CSIRT Engineer]
        │
        ↓ 同日中
[報告: CSIRT Team Leader]
        │
        ↓ 日次報告に含める
[経営報告: CISO（サマリー）]
```

### 3.4 P3（Low）フロー

```
[検知: SOC Analyst]
        │
        ↓ 対応・クローズ
[記録: SOC Analyst]
        │
        ↓ 週次報告に含める
[報告: CSIRT Team Leader]
```

## 4. 通知基準

### 4.1 即時通知が必要な状況

| 状況 | 通知先 | 理由 |
|------|--------|------|
| 個人情報漏洩確認 | CEO, CISO, CLO | 法令報告義務、経営判断 |
| サービス全面停止 | CEO, CISO | ビジネス影響、顧客影響 |
| ランサムウェア感染 | CEO, CISO, CLO, CFO | 身代金対応、復旧判断 |
| 内部関係者による不正 | CISO, CLO, HR | 懲戒処分、法的対応 |
| APT/国家関与攻撃疑い | CEO, CISO | 外部通報、政府連携 |
| メディア報道 | CEO, CMO | 広報対応 |

### 4.2 エスカレーション判断基準

```
以下のいずれかに該当する場合、上位へエスカレーション:

□ 自分の権限で判断・対応できない
□ リソース（人員・予算）が不足
□ 対応時間が規定を超過しそう
□ 影響範囲が当初想定より拡大
□ 攻撃が継続・激化している
□ 外部への影響が発生
□ 法的・コンプライアンス問題
□ メディア対応が必要
```

## 6. エスカレーション手順

### 6.1 エスカレーション実施手順

```
1. 状況評価
   □ インシデントの重大度を判定
   □ 影響範囲を確認
   □ エスカレーション先を決定

2. 情報整理
   □ インシデント概要（5W1H）
   □ 現在の対応状況
   □ 必要な判断・リソース
   □ 推奨アクション

3. 連絡実施
   □ 主連絡手段で連絡
   □ 応答がない場合、副連絡手段使用
   □ それでも応答がない場合、次の代理者へ

4. 記録
   □ エスカレーション日時
   □ 連絡先・連絡手段
   □ 伝達内容
   □ 受領確認
```

### 6.2 エスカレーション時の報告テンプレート

```
【インシデントエスカレーション報告】

■ インシデントID: INC-YYYY-NNNN
■ 重大度: P0/P1/P2/P3
■ カテゴリ: DATA/AVAIL/MALWARE/INSIDER/VULN

■ 概要:
[インシデントの簡潔な説明]

■ 発生日時: YYYY-MM-DD HH:MM (JST)
■ 検知日時: YYYY-MM-DD HH:MM (JST)

■ 影響:
- 影響システム:
- 影響データ:
- 影響ユーザー数:

■ 現在の状況:
[対応状況]

■ 必要な判断/リソース:
[エスカレーション理由、求める判断・支援]

■ 推奨アクション:
[提案する対応方針]

■ 報告者: [氏名]
■ 連絡先: [電話/メール]
```

## 8. 代理者・代替経路

### 8.1 代理者指定

| 役職 | 第1代理者 | 第2代理者 |
|------|-----------|-----------|
| CEO | CISO | CFO |
| CISO | CSIRT Team Leader | CTO |
| CLO | 外部顧問弁護士 | CISO |
| CFO | 経理部長 | CEO |
| CMO | 広報マネージャー | CEO |
| CSIRT Team Leader | CSIRT Engineer（シニア） | CISO |

### 8.2 連絡不通時の対応

```
1. 主担当者に3回連絡試行（各手段）
2. 5分経過で第1代理者に連絡
3. さらに5分経過で第2代理者に連絡
4. それでも不通の場合、上位階層に判断を仰ぐ

※ P0の場合は並行して全ての連絡手段を使用
```

## 9. 特殊ケースのエスカレーション

### 9.1 役員・経営層関与のインシデント

```
[検知: SOC Analyst]
        │
        ↓ 直接
[報告: 外部顧問弁護士] ←── CLOを経由しない
        │
        ↓
[対応: 取締役会/監査役]
```

```
[検知: 各拠点SOC]
        │
        ↓ 集約
[グローバルCSIRT（本社）]
        │
        ↓
[グローバルCISO]
        │
        ├──→ [各拠点への指示]
        └──→ [グローバルCEO報告]
```

### 9.3 外部委託先でのインシデント

```
[報告: 委託先担当者]
        │
        ↓
[受領: 委託管理担当]
        │
        ↓
[評価: CSIRT Team Leader]
        │
        ↓ 影響度に応じて
[通常エスカレーションフロー]
```
