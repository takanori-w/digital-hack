## 1. エグゼクティブサマリー

### 1.1 プロジェクト概要
LifePlan Navigatorは、個人のライフプラン支援を行うWebアプリケーションです。収入、資産、家族構成などの機微な個人情報を扱う金融系サービスとして、高度なセキュリティ対策が必要です。

### 1.2 脅威レベル評価
| 指標 | 評価 |
|------|------|
| **全体脅威レベル** | **HIGH (高)** |
| 攻撃可能性 | 非常に高い |
| 影響度 | 重大 |
| 攻撃者の関心度 | 高い |

### 1.3 主要な脅威サマリー
1. **クレデンシャルスタッフィング/アカウント乗っ取り** - 金融系アプリへの攻撃が250%増加（2024年）
2. **APIセキュリティの脆弱性** - Fintech APIへの攻撃が53%増加
3. **ランサムウェア攻撃** - 金融業界の67%が2024年に被害
4. **AI強化型フィッシング** - 60%の被害者がAI生成フィッシングに騙される

### 6.4 技術的対策詳細

```yaml
推奨事項:
  - WebAuthn/パスキー導入（フィッシング耐性MFA）

  - 認証試行制限: 5回/10分
```

#### API保護

```yaml
推奨事項:
  - OAuth 2.0/OpenID Connect実装
  - APIゲートウェイ導入
  - レート制限: 100リクエスト/分/ユーザー
  - 入力検証・サニタイゼーション
  - BOLA（Broken Object Level Authorization）対策
  - APIバージョニング・廃止管理
```

```yaml
推奨事項:
  - 保存時暗号化: AES-256

  - データマスキング（表示時）
```

| **API** | 異常なAPIコール頻度 | 通常の300%以上 | P1 |
| **API** | 認可エラー急増 | 50件/分 | P1 |
| **データ** | 大量データエクスポート | 1000件以上/時 | P0 |
| **セッション** | セッション乗っ取り兆候 | IP/UAの急変 | P1 |

### 7.2 検知ルール例（SIEM）

```sql
-- クレデンシャルスタッフィング検知
SELECT source_ip, COUNT(DISTINCT username) as attempts
FROM auth_logs
WHERE event_type = 'login_failed'
  AND timestamp > NOW() - INTERVAL '10 minutes'
GROUP BY source_ip
HAVING COUNT(DISTINCT username) > 10
```

## 8. CISO・SOCとの連携事項

### 8.1 CISO向け報告事項

1. **リスク受容判断が必要な項目**
   - MFA必須化のユーザビリティ影響
   - レート制限による正規ユーザーへの影響
   - セキュリティ投資の優先順位

2. **ポリシー策定が必要な項目**
   - インシデント対応手順
   - データ保持・削除ポリシー
   - サードパーティセキュリティ基準

### 8.2 SOC Analyst向け引継ぎ

1. **監視開始項目**
   - クレデンシャルスタッフィングパターン
   - API異常アクセス
   - データ窃取兆候

2. **IOC（Indicators of Compromise）**
   - 既知のクレデンシャルスタッフィングIPレンジ
   - 金融系マルウェアのシグネチャ
   - フィッシングドメインリスト
