## 1. エグゼクティブサマリー

### 1.1 背景

本監査計画書は、日本の上場企業として金融商品取引法に基づく内部統制報告制度（J-SOX）への準拠を確保しつつ、グローバルに展開する海外拠点（北アメリカ、南アメリカ、アジア、オーストラリア、ヨーロッパ、アフリカ、中東）を包括的に監査するための計画を定めるものである。

### 1.2 監査の必要性

- 日本上場企業としてのコンプライアンス義務
- グローバルサプライチェーンにおけるセキュリティリスクの増大
- 各国のデータプライバシー法への対応要件
- サイバー攻撃の高度化と国際的な脅威の増加

### 1.3 期待される成果

- 全社的なセキュリティガバナンス態勢の可視化
- 法規制準拠状況のギャップ分析
- リスクベースの改善ロードマップ策定
- 経営層への明確な報告と意思決定支援

## 2. 監査目的と範囲

### 2.1 監査目的

| 目的カテゴリ | 具体的目的 |
|------------|-----------|
| **コンプライアンス** | 各国法規制・業界標準への準拠状況確認 |
| **リスク評価** | 情報セキュリティリスクの特定・評価・優先順位付け |
| **統制評価** | 既存セキュリティ統制の有効性検証 |

## 4. 地域別監査要件

### 4.1 日本本社（親会社）

#### 適用法規制
- 個人情報保護法（2022年改正対応）
- 金融商品取引法（J-SOX）
- 不正競争防止法（営業秘密保護）
- サイバーセキュリティ基本法
- 経済安全保障推進法

#### 監査重点項目
1. J-SOX IT全般統制の有効性
2. 個人情報の安全管理措置
3. 海外子会社への統制伝達状況
4. グループセキュリティポリシーの整備状況
5. インシデント報告体制（72時間以内報告義務）

### 4.2 北アメリカ地域

#### 4.2.1 米国

**適用法規制**
- CCPA/CPRA（カリフォルニア州）
- 各州プライバシー法（Virginia、Colorado、Connecticut、Utah等）
- SOX（上場関連）
- 業種別規制（HIPAA、GLBA、FERPA等）
- SEC サイバーセキュリティ開示規則

**監査重点項目**
1. 消費者権利対応（アクセス権、削除権、オプトアウト）
2. サービスプロバイダー契約の適切性
3. データブローカー登録義務
4. サイバーインシデント開示（4営業日以内）
5. 州法間の相違への対応

#### 4.2.2 カナダ

**適用法規制**
- PIPEDA（連邦法）
- 州法（Quebec Bill 64、Alberta PIPA等）

**監査重点項目**
1. 有意な同意の取得
2. ブリーチ通知義務
3. 越境データ移転

#### 4.2.3 メキシコ

**適用法規制**
- Ley Federal de Protección de Datos Personales (LFPDPPP)

**監査重点項目**
1. プライバシーノーティス要件
2. ARCO権（アクセス、修正、削除、反対）
3. 国外移転要件

### 4.3 南アメリカ地域

#### 4.3.1 ブラジル

**適用法規制**
- LGPD（Lei Geral de Proteção de Dados）

**監査重点項目**
1. DPO（Data Protection Officer）任命
2. RIPD（データ処理記録）
3. ANPDへの通知義務
4. 国際データ移転の適法化根拠

#### 4.3.2 アルゼンチン

**適用法規制**
- Ley de Protección de Datos Personales (LPDP)

**監査重点項目**
1. データベース登録義務
2. 越境移転制限（EU適十分性認定国）

#### 4.3.3 チリ

**適用法規制**
- Ley 19.628（データ保護法）
- 新データ保護法案（審議中）

**監査重点項目**
1. 現行法と新法案の差分対応準備
2. 同意要件

### 4.4 ヨーロッパ地域

#### 4.4.1 EU全域

**適用法規制**
- GDPR（General Data Protection Regulation）
- NIS2指令（2024年10月施行）
- DORA（金融機関向け）
- AI Act（2024年施行開始）

**監査重点項目**
1. 処理の適法根拠（6条）
2. データ主体権利対応（15-22条）
3. DPIA実施状況（35条）
4. 国際移転メカニズム（SCC、BCR）
5. NIS2対応（セキュリティ要件強化）
6. DPO任命・届出

#### 4.4.2 英国（Brexit後）

**適用法規制**
- UK GDPR
- Data Protection Act 2018
- UK NIS Regulations

**監査重点項目**
1. UK代表者任命（EU外事業者）
2. UK-EU間データ移転（UK適十分性認定）
3. UK独自の規制動向

#### 4.4.3 ドイツ

**追加適用法規制**
- BDSG（連邦データ保護法）
- IT-Sicherheitsgesetz 2.0

**監査重点項目**
1. 労働者データ保護

#### 4.4.4 フランス

## 6. 監査スケジュール

### 6.1 全体スケジュール概要

```
Phase 1: 計画・準備
├── キックオフ・範囲確定
├── 法務確認（QA回収）
├── ツール・環境準備
└── 事前資料収集

Phase 2: 現地監査実行
├── Wave 1: 日本本社
├── Wave 2: アジア地域
├── Wave 3: ヨーロッパ地域
├── Wave 4: 北アメリカ地域
├── Wave 5: オーストラリア地域
├── Wave 6: 中東地域
├── Wave 7: アフリカ地域
└── Wave 8: 南アメリカ地域

Phase 3: 報告・フォローアップ
├── 発見事項分析・報告書作成
├── 経営報告・是正計画策定
└── フォローアップ監査計画
```

### 6.2 Wave別詳細

```
リスクベースアプローチ
│
├── 固有リスク評価
│   ├── 地域リスク（法規制、地政学）
│   ├── 事業リスク（業種、取扱データ）
│   └── 技術リスク（システム複雑性）
│
├── 統制リスク評価
│   ├── 統制設計の適切性
│   └── 統制運用の有効性
│
└── 残余リスク評価
    └── 改善優先順位付け
```

| **ウォークスルー** | 業務フロー追跡確認 | プロセス統制評価 |

## 10. リスク評価基準

### 10.1 発見事項の重要度分類

| 重要度 | 定義 | 対応期限目安 |
|--------|------|-------------|
| **Critical** | 直ちに重大なセキュリティ侵害につながる脆弱性、または法規制違反による即時の制裁リスク | 即時対応 |
| **High** | 重大なリスクをもたらす可能性のある脆弱性、または規制違反の高いリスク | 30日以内 |
| **Medium** | 中程度のリスク、追加の統制で軽減可能 | 90日以内 |
| **Low** | 軽微なリスク、ベストプラクティスからの逸脱 | 180日以内 |
| **Informational** | 改善の余地があるが直接的なリスクは低い | 次回監査まで |

### 10.2 リスク評価マトリクス

```
影響度
  ▲
高│ Medium  │  High   │ Critical│
  │─────────┼─────────┼─────────│
中│   Low   │ Medium  │  High   │
  │─────────┼─────────┼─────────│
低│Info     │   Low   │ Medium  │
  └─────────┴─────────┴─────────▶ 発生可能性
        低       中       高
```
