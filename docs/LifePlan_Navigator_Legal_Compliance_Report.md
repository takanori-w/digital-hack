## 2. 個人情報保護法対応の検証

### 2.1 収集データの分析

**現在収集しているデータ（types/index.ts より）:**

```typescript
interface UserProfile {
  id: string;           // 識別子
  name: string;         // 氏名 ← PII
  email: string;        // メールアドレス ← PII
  birthDate: string;    // 生年月日 ← PII
  gender: string;       // 性別
  prefecture: string;   // 都道府県 ← PII（居住地）
  city: string;         // 市区町村 ← PII（居住地）
  occupation: string;   // 職業
  annualIncome: number; // 年収 ← センシティブ情報
  householdSize: number;    // 世帯人数
  maritalStatus: string;    // 婚姻状況
  hasChildren: boolean;     // 子供の有無
  numberOfChildren: number; // 子供の人数
  childrenAges: number[];   // 子供の年齢 ← センシティブ情報
  housingType: string;      // 住居形態
}
```

### 2.3 データフロー分析

```
[ユーザー入力]
     │
     ▼
[フロントエンド（ブラウザ）]
     │ Zustand persist → localStorage
     ▼
[ローカルストレージ] ← 現状ここで保存（サーバー送信なし）
```

**現状の評価:**
- ✅ デモ版のためサーバーへの送信なし（localStorage保存のみ）
- ⚠️ 本番環境では要対応（サーバー保存時は個人情報保護法完全対応必須）
- ⚠️ localStorage保存でも個人情報であることに変わりなし

## 3. 金融商品取引法上の免責事項確認

### 3.1 法的リスク分析

**金融商品取引法上の懸念事項:**

| 行為類型 | 該当可能性 | 現状対応 | リスク |
|----------|-----------|----------|--------|
| 投資助言業 | 低〜中 | 一部免責あり | 中 |
| 投資運用業 | なし | - | なし |
| 金融商品仲介業 | なし | - | なし |

**投資助言業該当性の分析:**

当サービスの機能:
1. 将来シミュレーション（3シナリオ）
2. iDeCo・NISA等の制度紹介
3. 投資効果の計算表示

**判断:**
- 一般的な制度情報の提供は投資助言に該当しない
- 個別具体的な投資推奨を行わない限り問題なし
- ただし、シミュレーション結果の表示方法に注意が必要

### 3.2 現在の免責表示の評価

**LandingPage.tsx（137行目）:**
```html
本サービスは情報提供を目的としており、金融商品の勧誘を目的とするものではありません。
```

**SimulationChart.tsx（219-222行目）:**
```html
※ このシミュレーションは参考値です。実際の運用結果は市場環境により異なります。
```

**評価:** 🟡 最低限の免責は記載されているが、不十分

### 3.3 必要な免責事項

#### 3.3.1 シミュレーション画面に追加すべき免責

```
【重要事項】
• 本シミュレーションは一定の前提条件に基づく試算であり、将来の運用成果を保証するものではありません。
• 実際の投資にはリスクが伴い、元本割れする可能性があります。
• 本サービスは金融商品取引法に基づく投資助言業の登録を受けておらず、特定の金融商品の購入・売却を推奨するものではありません。
• 投資判断は、ご自身の責任において行ってください。
• 税制・法制度は変更される可能性があります。最新情報は各機関の公式サイトでご確認ください。
```

#### 3.3.2 補助金・制度情報画面に追加すべき免責

```
【ご注意】
• 掲載情報は作成時点のものであり、最新の情報と異なる場合があります。
• 制度の詳細・申請条件は、各実施機関の公式情報をご確認ください。
• 申請の可否や支給額は個人の状況により異なります。
```
